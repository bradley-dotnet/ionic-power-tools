Dynamic Router

When moving or refactoring routes, it's easy to forget to update a link or navigation command in the application. Dynamic Routing changes hard-coded links to compile-time enums that are then mapped to correct URLs at runtime, making broken links a thing of the past. It also enables better validation of route parameters prior to navigation and provides a cleaner API to routing in general.

Routing Table Setup

  1. Create an enum with values representing each route you would like to navigate to: navigation-targets.enum 
    export enum NavigationTargets {{'{'}}
    Home,
    Tasks,
    Details
}
  2. Import DynamicRoute into your routing module: app-routing.module.ts 
    import {{'{'}} DynamicRoute } from 'ionic-power-tools';
  3. Change your existing Routes array definition to instead use DynamicRoute<NavigationTargets>[]: app-routing.module.ts 
    const routes: DynamicRoute<NavigationTargets>[] = [ ... ];
  4. For each eagerly route with a path defined, add a navigationTarget assignment to the corresponding enum value: app-routing.module.ts 
    const routes: DynamicRoute<NavigationTargets>[] = [
    {{'{'}}
        path: '',
        redirectTo: 'home',
        pathMatch: 'full'
    },
    {{'{'}}
        path: 'home',
        component: HomeComponent,
        navigationTarget: NavigationTargets.Home
    },
    {{'{'}}
        path: 'tasks',
        component: TasksComponent,
        navigationTarget: NavigationTargets.Tasks
    },
    ...
];

Navigation From HTML Templates

  1. Add an enum property to the component to enable enum value binding: app.component.ts 
    import {{'{'}} NavigationTargets } from './navigation-targets.enum';

@Component({{'{'}}...})
export class AppComponent {{'{'}}
    ...

    public NavigationTargets = NavigationTargets;
}
  2. Replace each routerLink directive and its hardcoded path with iptDynamicRouterLink and an enum value: app.component.html 
    <ion-item [button]="true" [iptDynamicRouterLink]="NavigationTargets.Tasks">
    <ion-label>Tasks</ion-label>
</ion-item>

Lazy Loaded Routes

  1. Move the routes array from the child-routing.module.ts into a file called child.routes.ts
  2. Define an enum with an entry for each component that will appear in the route configuration child.routes.ts 
    export enum ChildComponents {{'{'}}
    ListComponent,
    DetailsComponent
}
  3. Redfine the array as a LazyChildren{{'\'}} child.routes.ts 
    export const childRoutes: {{'LazyChildren\'}} = [
  4. Replace all component field values with the corresponding enum member child.routes.ts 
    {{'{'}}
    path: '',
    component: ChildComponents.ListComponent,
    children: [
        {{'{'}}
            path: ':itemId',
            component: ChildComponents.DetailsComponent
        }
    ]
},
  5. In the feature routing module, use the provided populateRoutes method in RouterModule.forChild child-routing.module.ts 
    {{'import { populateComponents \} from "ionic-power-tools";'}}
{{'import { ListComponent } from "./list.component";'}}
{{'import { DetailsComponent } from "./details.component";'}}
{{'import { ChildComponents, childRoutes } from "./child.routes";'}}

const componentMap: Record{{'\\> = {'}}
    [ChildComponents.List]: ListComponent,
    [ChildComponents.Details]: DetailsComponent
{{'}'}}


{{'@NgModule({'}}
    imports: [RouterModule.forChild(populateComponents(componentMap, childRoutes))],
    exports: [RouterModule]
{{'})'}}
export class ChildRoutingModule {{'{'}} {{'}'}}
  6. From the root routing module, lazy loaded routes are the same as usual but with childRouteConfigs specified, pointing to the unpopulated route config app-routing.module.ts 
    ...
{{'{'}}
    path: 'child',
    loadChildren: () => import('./child/child-routing.module').then(m => m.ChildRoutingModule),
    childRouteConfigs: childRoutes,
}
